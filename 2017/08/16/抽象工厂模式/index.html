抽象工厂模式

发表于 2017-08-16 | 分类于 设计模式

背景介绍

在工厂模式中，我们将不同类对象的获取交由不同的工厂类来实现，这样导致的一个问题就是，对于每一个类都要设计一个对应的工厂类，使得工厂类的数量很多，不便于维护，增加了开发成本，而抽象工厂模式则对这一问题进行了改进。在工厂模式中，一个工厂类中只提供一种类对象的获取方法；但在抽象工厂模式中，一个工厂类提供了多个类对象的获取方法，可以生产一组业务上相关的对象。

抽象工厂模式(Abstract Factory Pattern)：提供一个创建一系列相关或相互依赖对象的接口，而无须指定它们具体的类。抽象工厂模式又称为Kit模式，它是一种对象创建型模式。

总结

抽象工厂对工厂模式进行了一些改进，这样可以减少具体工厂类的数量，如果新增一组业务逻辑例如要生产绿色三角形，则只需要新增绿色和三角形两个类，而不需要对原来的代码进行修改，但如果要红色铁质的正方形则需要修改原先的工厂类代码，这也违背了“开闭原则”。总之，抽象工厂模式是对一组业务逻辑的抽象，每一个工厂对应一组产品。
