原型模式

发表于 2017-08-16 | 分类于 设计模式

背景介绍

在使用电脑时，我们经常使用到复制粘贴的功能，即通过这种方式，生成一个和被复杂对象一样的东西。还有一个大家比较熟悉的就是西游记中的孙悟空能通过头发变化出很多一摸一样的孙悟空。原型模式就是这种目的，通过原型复制出很多和原型一样的对象。

原型模式(Prototype Pattern)：使用原型实例指定创建对象的种类，并且通过拷贝这些原型创建新的对象。原型模式是一种对象创建型模式。

下面简单介绍下Java的浅拷贝和深拷贝：

浅拷贝一般针对类的成员变量全为基本数据类型，通过简单的赋值操作便可以实现拷贝，但如果类的成员变量是一个其他的类，赋值得到的是类的引用，如果对该变量进行修改是会改变原型成员变量的值。此时需要进行深拷贝，深拷贝的方法是使用序列化和反序列化实现的。序列化就是将对象写到流的过程，写到流中的对象是原有对象的一个拷贝，而原对象仍然存在于内存中。通过序列化实现的拷贝不仅可以复制对象本身，而且可以复制其引用的成员对象，因此通过序列化将对象写到一个流中，再从流里将其读出来，可以实现深克隆。需要注意的是能够实现序列化的对象其类必须实现Serializable接口，否则无法实现序列化操作。
