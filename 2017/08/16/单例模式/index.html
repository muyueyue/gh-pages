单例模式

发表于 2017-08-16 | 分类于 设计模式

背景介绍

在我们的软件设计当中有时会遇到一种问题，即某个对象或资源在系统中是唯一存在的，不会出现多个。就比如在一个王国当中国王只有一个。因此对于国王类我们只能对它实例化一次。

单例模式(Singleton Pattern)：确保某一个类只有一个实例，而且自行实例化并向整个系统提供这个实例，这个类称为单例类，它提供全局访问的方法。单例模式是一种对象创建型模式。

代码实现

单例模式有多种实现方式

  • 懒汉式，线程不安全

所谓懒汉，就是等到要用的时候才初始化对象–new Singleton();可以和后面的饿汉模式对比就明白了。

  • 懒汉式，线程安全

对于上面这种模式，存在一个问题，每次只有一个对象能调用getInstance()方法。但对于instance != null的情况下是不需要进行现场同步的，同步只需要用在程序第一次获取单例的时候，于是引出了下面这种模式。

  • 双重检验锁

上面这种模式在instance != null的情况下，我们不会进入到同步块，即在程序第一次获取单例后，以后的获取都不会被同步，提高了程序执行的速度。

  • 饿汉式

从上面可以清楚的看出，在类加载带虚拟机的时候就对单例进行了初始化。因此称其为饿汉，迫不及待的要得到单例。

饿汉模式是线程安全的。

  • 静态内部类式
  • 枚举式

创建枚举默认就是线程安全的，所以不需要担心double checked locking，而且还能防止反序列化导致重新创建新的对象。

总结

单例模式的实现有多种，根据实际情况使用。单例也不能完全保证程序中只有唯一一个对象，例如使用反射机制也是可以实例化出多个对象。因为反射可以调用类的私有方法，即可以直接调用单例的构造方法进行实例化。
